# Internationalization (i18n)

The frontend (price list and product detail pages) supports **English** (default) and **Dutch**.

## Language selection

- In **Admin → General settings** set **Language (frontend)** to **English** or **Dutch** (Nederlands).
- The chosen language applies to the public price list and product pages. The admin interface keeps the browser language.

## Translation files

- **Location:** `locale/<lang>/LC_MESSAGES/django.po` (and compiled `django.mo`).
- **Languages:** `en` (English), `nl` (Dutch).
- Source strings in code and templates are in **English** (msgid). Dutch translations are in `locale/nl/LC_MESSAGES/django.po`.

## Updating translations

1. Edit the `.po` files (e.g. add or change `msgstr` in `locale/nl/LC_MESSAGES/django.po`).
2. **Compile to `.mo`** (Django uses `.mo` files; without this step the chosen language will not show translated strings):
   - **Option A:** If you have [GNU gettext](https://www.gnu.org/software/gettext/) installed:  
     `python manage.py compilemessages`
   - **Option B (e.g. Windows without gettext):** Using Babel:  
     `pip install babel`  
     `pybabel compile -d locale -D django`
3. Restart the dev server if it is running.

## Adding new translatable strings

- In **templates:** wrap text in `{% trans "Your English text" %}` and load i18n: `{% load i18n %}`.
- In **Python:** use `from django.utils.translation import gettext as _` and `_("Your English text")`.
- Then update the `.po` files (run `python manage.py makemessages -l nl` if gettext is installed, or add the new msgid/msgstr by hand) and recompile.
